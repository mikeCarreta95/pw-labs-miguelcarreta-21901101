Report PW-Lab1 💻

Com o inicio da unidade curricular Programação Web, foi nos pedido que criássemos uma página Web e consequentemente reportássemos tudo na atual página que vos apresento.



  1. Alojamento de página Web na cloud

    • Criação de uma página Web

      • Foi nos disponibilizado um código fonte HTML.

    • Criação de repositório GitHub

      • Foi nos pedido que criássemos um repositório no GitHub.

    • Alojamento no Heroku

      • Por conseguinte, tivemos que criar uma conta no Heroku e sincroniza-la com o GitHub de maneira a ser possivel alojar a página Web na cloud.


  2. Conhecer a Internet

    • Endereços IP

      • Foi pedido que apresentássemos o IP do nosso Computador, Telemovél e do servidor do Heroku onde se encontra a nossa App. Começo por mostrar aqui em baixo o Ip do Computador e Telemovél


    • Percurso

      • Usámos o comando <tracert> para rastrear a rota pelos quais os pacotes passam até o seu destino, indicando o tempo que demoram por "salto" entre router.


      De seguida, usei o GeoTraceroute para visualizar graficamente por onde passam os pacotes IP, até chegar ao seu servidor Heroku.


      É claro que existem 3 "saltos", sendo eles:

      • Prior velho
        • ↓
      • Lisboa
        • ↓
      • Frankfurt

  3. Acesso via HTTP à minha página Web

    • HTTP GET

      • O protocolo de troca de mensagens entre um cliente e um servidor Web é o HTTP. Com o atalho Ctrl + U podes averiguar que a resposta do servidor é o mesmo ficheiro que foi escrito primeiramente.


    • Inspect

      • Usando o atalho Ctrl + Shift + i podemos inspeccionar ficheiros descarregados pelo browser, permitindo analisar uma grande variedade de informação.

      Dentro dessa ferramenta podemos encontrar informações tais como:

      PREVIEW
      Pré-visualizar o conteúdo do ficheiro.
      HEADERS
      Informação pedida ao servidor sobre o ficheiro.
      TIMING
      Informação sobre a linha temporal do pedido.
